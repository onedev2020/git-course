Практическая работа
2.1 Создание проекта, репозитория и коммита

• Создать папку проекта (c:\repo\myproject)
• В папке проекта создать пару текстовых файлов
• В bash перейти в папку проекта (cd /c/repo/ myproject)
• Проверить содержимое папки (ls)
• Создать репозиторий (git init)
• Добавить файлы в stage (git add readme.txt)
• Проверить stage (git status)
• Создать коммит (git commit)
• Проверить историю (git log)

2.2. Работа с коммитами
Создать коммит (использовать сначала git status и git diff,
потом git commit)
• Дополнить коммит с помощью git commit --amend
• Посмотреть историю коммитов в разных вариантах git log
• Восстановить отредактированный файл git restore
• Удалить файл (git rm)
• Переименовать файл (git mv)
• Сделать коммит
• Создать тэг на последний коммит
